---
title: 'List Tasks'
openapi: 'GET /tasks'
description: 'Retrieve all tasks, optionally filtered by status'
icon: 'list'
---

<Info>
  Returns all tasks the authenticated user has access to. Use query parameters to filter by status or limit the number of results.
</Info>

## Usage Notes

- Results are returned as a JSON array of task objects.
- Each task includes `title`, `status`, and an optional `priority` field.
- Filter by status using `?status=pending`, `?status=in_progress`, or `?status=completed`.
- If no tasks exist, the endpoint returns an empty array `[]`.

### Example Response

```json
[
  {
    "id": 1,
    "title": "Set up MCP filesystem server",
    "status": "completed",
    "priority": "high"
  },
  {
    "id": 2,
    "title": "Write OpenAPI spec for webhook handler",
    "status": "in_progress",
    "priority": "medium"
  }
]
```
