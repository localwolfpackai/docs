This section is a **working template** for API documentation. I use it as a starting point whenever I need to document an API — whether it's a personal project, a webhook handler, or a microservice.

The example below documents a simple **Tasks API** — a pattern I use for agent-driven workflows where tasks are created, tracked, and completed programmatically.

## How It Works

Mintlify generates interactive API docs from an [OpenAPI spec](https://mintlify.com/docs/api-playground/openapi/setup). You write the spec, create thin MDX wrappers for each endpoint, and the framework handles the rest — request/response schemas, auth headers, and the try-it-out playground.

## Base URL

```
https://api.example.com
```

## Endpoints

| Method | Path | What it does |
|--------|------|-------------|
| `GET` | `/tasks` | List all tasks |
| `POST` | `/tasks` | Create a new task |
| `DELETE` | `/tasks/{id}` | Delete a task by ID |

## Webhooks

| Event | Path | What it does |
|-------|------|-------------|
| Task Created | `/tasks/webhook` | Fires when a new task is created |

## Auth

Everything uses Bearer tokens. See the [Authentication](/api-reference/authentication) page.

## Errors

All errors return JSON with `error` (status code) and `message`:

```json
{
  "error": 400,
  "message": "What went wrong"
}
```
