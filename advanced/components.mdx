## MCP Servers

Model Context Protocol servers give agents access to external tools and data. They're the bridge between the agent and the real world.

### How MCP Works

An MCP server exposes **tools** (actions the agent can call), **resources** (data the agent can read), and **prompts** (reusable templates). The agent discovers what's available and decides when to use them.

```json claude_desktop_config.json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/dir"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "ghp_..." }
    }
  }
}
```

### Servers I Run

| Server | What it does | When I use it |
|--------|-------------|---------------|
| **Filesystem** | Read/write/search files on disk | Always on — core agent capability |
| **GitHub** | Create PRs, manage issues, search repos | Shipping code, reviewing PRs |
| **Fetch** | Make HTTP requests from agent context | Checking APIs, pulling docs |
| **Memory** | Persist knowledge across sessions | Long-running projects |

### Writing a Custom MCP Server

If you need something not covered by the official servers, you can write your own. The protocol is straightforward:

```typescript server.ts
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import { z } from "zod";

const server = new McpServer({
  name: "my-server",
  version: "1.0.0",
});

server.tool(
  "get_weather",
  "Get current weather for a city",
  { city: z.string() },
  async ({ city }) => {
    const data = await fetchWeather(city);
    return { content: [{ type: "text", text: JSON.stringify(data) }] };
  }
);

const transport = new StdioServerTransport();
await server.connect(transport);
```

<Tip>
  Start with the official MCP servers and only build custom ones when you have a specific need. Most common use cases are already covered.
</Tip>

## Skills.sh

[Skills.sh](https://skills.sh) is a package manager for agent knowledge. Instead of writing the same `CLAUDE.md` instructions for every project, you install skills that inject context automatically.

```bash
# add a skill to your project
npx skills add claude-code

# see what's available
npx skills search "react"

# list what you have installed
npx skills list
```

### How Skills Work

A skill is a markdown file that gets injected into the agent's context. It can contain:

- Coding conventions and patterns
- Framework-specific knowledge
- Testing strategies
- Common pitfalls and fixes

Think of it as community-maintained `CLAUDE.md` fragments that you compose together.

### Making Your Own Skill

If you have project-specific patterns you reuse across repos, you can publish them as a skill:

```bash
npx skills create my-pattern
npx skills publish
```
