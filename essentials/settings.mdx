Everything about this site is controlled by one file: `docs.json`. Here's what each section does.

## Core Properties

<ResponseField name="name" type="string" required>
  The site name. Shows in the browser tab and various metadata.

  Example: `"humanlup docs"`
</ResponseField>

<ResponseField name="theme" type="string">
  The base theme. I use `"mint"`.
</ResponseField>

<ResponseField name="favicon" type="string">
  Path to the favicon file.

  Example: `"/favicon.svg"`
</ResponseField>

## Colors

<ResponseField name="colors" type="Colors" required>
  Hex color codes for the site theme.
  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      The main accent color used for highlighted content, links, and section headers.
    </ResponseField>
    <ResponseField name="light" type="string">
      Accent color override for dark mode.
    </ResponseField>
    <ResponseField name="dark" type="string">
      Color used for primary buttons.
    </ResponseField>
    <ResponseField name="background" type="object">
      Custom background colors for light and dark mode.
      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          Background color in light mode.
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          Background color in dark mode.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

Here's what I use — neutral slate with a muted blue accent:

```json
{
  "colors": {
    "primary": "#64748B",
    "light": "#94A3B8",
    "dark": "#475569"
  }
}
```

## Logo

<ResponseField name="logo" type="string or object">
  Path to a single logo image, or an object with separate light/dark variants.
  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Logo shown in light mode.
    </ResponseField>
    <ResponseField name="dark" type="string">
      Logo shown in dark mode.
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      Where clicking the logo navigates to.
    </ResponseField>
  </Expandable>
</ResponseField>

```json
{
  "logo": {
    "light": "/logo/light.svg",
    "dark": "/logo/dark.svg"
  }
}
```

## Navigation

The `navigation` field controls the entire sidebar structure. It uses a tabs-and-groups model.

<ResponseField name="navigation" type="Navigation" required>
  The full site navigation tree.
  <Expandable title="Navigation">
    <ResponseField name="tabs" type="Tab[]">
      Top-level tabs (e.g., "Toolkit", "API Reference").
      <Expandable title="Tab">
        <ResponseField name="tab" type="string">
          The display label for this tab.
        </ResponseField>
        <ResponseField name="groups" type="Group[]">
          Groups of pages within this tab.
          <Expandable title="Group">
            <ResponseField name="group" type="string">
              The group heading shown in the sidebar.
            </ResponseField>
            <ResponseField name="pages" type="string[]">
              Page paths (without `.mdx` extension).
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="global" type="object">
      Global navigation elements that appear on every tab.
      <Expandable title="Global">
        <ResponseField name="anchors" type="Anchor[]">
          External links shown as anchors in the sidebar.
          <Expandable title="Anchor">
            <ResponseField name="anchor" type="string">
              The display label.
            </ResponseField>
            <ResponseField name="href" type="string">
              The URL to link to.
            </ResponseField>
            <ResponseField name="icon" type="string">
              Font Awesome icon name.
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

Here's a simplified version of what I use:

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "Toolkit",
        "groups": [
          {
            "group": "Start Here",
            "pages": ["index", "quickstart", "development"]
          }
        ]
      },
      {
        "tab": "API Reference",
        "groups": [
          {
            "group": "Overview",
            "pages": ["api-reference/introduction"]
          }
        ]
      }
    ]
  }
}
```

<Tip>
  Page paths don't include the `.mdx` extension. The order in the array determines the display order.
</Tip>

## Navbar

<ResponseField name="navbar" type="Navbar">
  Controls the top navigation bar.
  <Expandable title="Navbar">
    <ResponseField name="links" type="Link[]">
      Text links shown in the navbar.
    </ResponseField>
    <ResponseField name="primary" type="object">
      The primary CTA button in the navbar. Set `type` to `"github"` to show repo stars.
    </ResponseField>
  </Expandable>
</ResponseField>

```json
{
  "navbar": {
    "links": [
      { "label": "GitHub", "href": "https://github.com/localwolfpackai" }
    ],
    "primary": {
      "type": "github",
      "label": "GitHub",
      "href": "https://github.com/localwolfpackai/docs"
    }
  }
}
```

## OpenAPI

<ResponseField name="openapi" type="string | string[]">
  Path(s) or URL(s) pointing to your OpenAPI spec file(s). Mintlify auto-generates API playground pages from these.

  Example: `["/api-reference/openapi.json"]`
</ResponseField>

## Footer & Socials

<ResponseField name="footer" type="Footer">
  Footer configuration including social links.
  <Expandable title="Footer">
    <ResponseField name="socials" type="object">
      Key-value pairs where the key is the platform name and the value is the URL.

      Supported keys: `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`
    </ResponseField>
  </Expandable>
</ResponseField>

```json
{
  "footer": {
    "socials": {
      "github": "https://github.com/localwolfpackai"
    }
  }
}
```

## Feedback

<ResponseField name="feedback" type="Feedback">
  Enables feedback buttons on every page.
  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
      Shows a "Suggest Edit" button that opens a PR.
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
      Shows a "Raise Issue" button.
    </ResponseField>
  </Expandable>
</ResponseField>

## Search

<ResponseField name="search" type="Search">
  Customize the search experience.
  <Expandable title="Search">
    <ResponseField name="prompt" type="string">
      Placeholder text in the search bar.
    </ResponseField>
  </Expandable>
</ResponseField>

## Dark Mode Toggle

<ResponseField name="modeToggle" type="ModeToggle">
  Customize or hide the dark/light mode toggle.
  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Force a default mode for new visitors.
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      Hide the toggle entirely. Combine with `default` to lock to one mode.
    </ResponseField>
  </Expandable>
</ResponseField>

## Versions

<ResponseField name="versions" type="string[]">
  Array of version names. Adds a version dropdown to the nav bar. I don't use this yet — see [Versioning](/advanced/versioning).
</ResponseField>

## Background Image

<ResponseField name="backgroundImage" type="string">
  A background image displayed behind every page. See [Infisical](https://infisical.com/docs) for an example.
</ResponseField>
