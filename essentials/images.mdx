## Image

### Using Markdown

The [markdown syntax](https://www.markdownguide.org/basic-syntax/#images) lets you add images using the following code:

```md
![title](/images/my-image.jpg)
```

Note that the image file size must be less than 5MB. Otherwise, we recommend hosting on a service like [Cloudinary](https://cloudinary.com/) or [S3](https://aws.amazon.com/s3/). You can then use that URL and embed.

### Using HTML Embeds

To get more customizability with images, you can also use HTML `<img>` tags directly:

```html
<img height="200" src="/images/my-image.jpg" alt="Description of the image" />
```

### Light and Dark Mode

You can display different images depending on the user's color scheme using CSS classes:

```html
<img className="block dark:hidden" src="/images/hero-light.png" alt="Light mode hero" />
<img className="hidden dark:block" src="/images/hero-dark.png" alt="Dark mode hero" />
```

<Tip>
  Store all images in the `/images/` directory and reference them using root-relative paths like `/images/filename.png`.
</Tip>

### Using the Frame Component

The `<Frame>` component adds a styled border and optional caption to images:

```html
<Frame caption="A beautiful landscape">
  <img src="/images/my-image.jpg" alt="Landscape" />
</Frame>
```

### iFrames

Loads another HTML page within the document. Most commonly used for embedding videos.

```html
<iframe src="https://www.youtube.com/embed/4KzFe50RQkQ"></iframe>
```

### Best Practices

- **Alt text**: Always include descriptive `alt` attributes for accessibility.
- **File size**: Keep images under 5MB. Compress PNGs and use SVGs for logos and icons.
- **File format**: Use SVG for vector graphics, PNG for screenshots, and JPG for photographs.
- **Dark mode**: Provide both light and dark variants for hero images and diagrams.
